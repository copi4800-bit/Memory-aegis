# Feature Spec: 087 Full Core UX Showcase

## Summary

Expose the full class of Aegis core strengths through one deliberate user experience instead of scattering them across separate operator tools.

## Problem

Aegis can already prove a lot of core strength through spotlight, governance, evidence, signals, and gauntlets, but the user still has to mentally assemble that story from multiple surfaces.

## Goals

- Create one first-class UX surface that shows the full Aegis memory-core story for a single query.
- Reuse existing core/runtime surfaces instead of inventing parallel logic.
- Make the strongest parts of Aegis legible in one run: truth selection, why-this, why-not, evidence, governance, signals, graph context, and health.

## Non-Goals

- Building a consumer GUI in this phase.
- Rewriting retrieval or governance logic.
- Replacing the smaller spotlight tool.

## Acceptance Criteria

1. The repo gains one full-core showcase surface.
2. The showcase is available through runtime/tooling, not just docs.
3. Tests prove the showcase surfaces the expected core sections.
